 ## Структура репозитория
 
```
.
├── tvil_hotels.py            # Шаг 1: парсинг списка отелей + каталог
├── tvil_rooms.py             # Шаг 2: парсинг номеров и цен
├── tvil_statistic.py         # Шаг 3: сводная статистика за день
├── log_config.py             # Логирование + Telegram-уведомления
├── daily/
│   ├── hotels/               # YYYY-MM-DD.csv — отели за день
│   ├── rooms/                # YYYY-MM-DD.csv — номера за день
│   └── statistics/           # YYYY-MM-DD.csv — статистика за день
├── catalog/
│   └── hotels.csv            # Накопительный каталог отелей
├── logs/                     # Логи запусков
└── .github/workflows/
    └── tvil-parser.yml       # Ежедневный запуск в GitHub Actions
```
 
## Форматы данных
 
### `daily/hotels/YYYY-MM-DD.csv`
 
| Колонка | Описание |
|---|---|
| `city` | Город |
| `tvil_hotel_id` | Идентификатор отеля в Tvil |
| `name` | Название отеля |
| `address` | Адрес |
| `latitude` / `longitude` | Координаты |
| `url` | Ссылка на страницу отеля |
| `rooms_number` | Общее число номеров в отеле |
 
### `daily/rooms/YYYY-MM-DD.csv`
 
| Колонка | Описание |
|---|---|
| `tvil_hotel_id` | Идентификатор отеля |
| `room_name` | Название/описание номера (из фотографий объекта) |
| `room_id` | Идентификатор номера |
| `free_rooms` | Кол-во свободных номеров этого типа |
| `all_rooms` | Всего номеров этого типа (из текста «Свободны X из Y») |
| `room_capacity` | Вместимость номера (из описания «N-местный») |
| `price` | Цена за период, ₽ |
| `url` | Ссылка на страницу отеля |
 
Если у отеля нет номеров в выдаче, в файл пишется строка-заглушка с пустым `room_id` — для контроля пропусков.
 
### `daily/statistics/YYYY-MM-DD.csv`
 
| Колонка | Описание |
|---|---|
| `tvil_hotel_id`, `name` | Отель |
| `date` | Дата сбора |
| `rooms_num` | Всего номеров в отеле |
| `free_rooms_amount` | Сумма `free_rooms` по всем типам номеров |
| `max_capacity` | Суммарная вместимость свободных номеров |
| `available_rooms_percent` | Доля свободных номеров, % |
| `min_price` | Минимальная цена (только среди номеров со свободными местами), ₽ |
 
### `catalog/hotels.csv`
 
Те же поля, что и в дневном файле отелей, плюс `first_seen_date` и `last_seen_date` — когда отель впервые и в последний раз встречался в выдаче.
